# ask-gemini (AI-Driven Development Orchestrator)

**ask-gemini** は、巨大なコードベース（数MB〜GB単位）における AI 駆動開発（AIDD）の「トークン上限超過・ハルシネーション・高額なAPIコスト」を解決するために設計された、独立型の Node.js スクリプト（プロキシ）です。

200万トークンの巨大なコンテキストを無料枠で扱える **Gemini 1.5 Pro (Google)** と、精緻なコーディングと自律解決能力に長けた **Claude Code (Anthropic)** の間を取り持つことで、完璧な役割分担を実現します。

## 💡 本ツールが解決する課題（Information Diet）

大規模なプロジェクト（ビルドファイルや `node_modules` などを含む）を直接 Claude Code 等のエージェントに読ませようとすると、一瞬でコンテキストウィンドウ（例: Claude 3.5 の 20万トークン）を食いつぶし、エラーによる停止や甚大なハルシネーションを引き起こします。

`ask-gemini` はこの問題を以下の**情報ダイエット（Information Diet）**と最適化アプローチで解決します：

1. **完全なステートレス・全ソース送信**: `git ls-files` と連動し、画像や巨大なロックファイル等のバイナリを除外した「人間が管理している純粋なテキストソース」のみ（約1.5MB以内）に絞り込みます。
2. **Gemini による俯瞰と執刀指示**: 絞り込んだ最新の全ソースを Gemini 1.5 Pro に一括で丸呑みさせ、全体構造を把握した上での「具体的な手術計画書（どのファイルをどう直すか）」を生成させます。
3. **Claude へのシームレスなパス**: Claude Code は、出力された短く的確な「手術計画書」だけを読み取り、数個のファイルへ向けたピンポイントな直接編集を行います。巨大なソースを一括で読み込む必要がなくなり、安全かつ低コストにタスクを完了できます。

## 🛡️ フェイルセーフ（エラー時の自律リカバリ機能）

Gemini 1.5 Pro の無料枠レート制限（HTTP 429 Error等）に達した場合、このスクリプトはパニックを起こしません。
代わりに、**Claude Code が解釈して自律実行できる特別な「フォールバックメッセージ」を出力して正常終了 (exit 0)** します。

これにより、Claude は「Geminiでの分析に失敗したため、自分の力（個別ファイルの探索や読み込み）だけで安全にタスクを完了しなければならない」と判断を切り替え、全自動での作業継続（あるいは人間への報告）を実現します。

## ⚙️ セットアップとキー管理

*   **依存関係ゼロ（Zero-Dependency）**: \`fs\`, \`child_process\`, \`https(fetch)\` のみを使用し、`package.json` すら不要です。
*   **セキュアなキー保存**: 初回起動時にターミナルで対話形式で Gemini API キーの入力を求めます。ハードコーディングを排除し、OSのホームディレクトリ（`~/.ask-geminirc.json`）にパーミッション `0o600` で永続保存します。

## 🚀 使い方

`ask-gemini.mjs` を対象の Git リポジトリのルートディレクトリに配置して実行します。

### 一般的な使用方法
```bash
node ask-gemini.mjs "カレンダーコンポーネントの表示崩れを治すための手術計画書を作成して"
```

### Claude Code での利用プロセス（プロンプト例）
> `カレンダーのバグを直したいので、まずターミナルで node ask-gemini.mjs "バグ修正の方針を作成して" を実行し、その標準出力の指示（手術計画）に厳密に従って、手元のファイルを編集してください。`

### CLI オプション
```bash
node ask-gemini.mjs --help          # ヘルプメニューの表示
node ask-gemini.mjs --reset-key     # 保存されたAPIキーの再設定
```
