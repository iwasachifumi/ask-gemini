## @gemini ショートカット
メッセージの文頭が `@gemini ` で始まる場合、以下のコマンドを自動実行する：
`node <ask-gemini.mjsの絶対パス> "<@gemini の後ろのテキスト>"`

### ⚠️ マルチモーダル（画像転送）の絶対ルール
もしユーザーのメッセージに「画像ファイル（スクリーンショット等）」や「PDFファイル」が添付されている場合、Claude 自身がそれを見るだけでなく、Gemini にも視覚情報を共有するために、必ずコマンドの末尾に `--attach` オプションを用いてファイルの絶対パスを渡して実行すること。

【コマンド生成例】
入力: 「@gemini このスクショのようにレイアウト崩れてるの直して（error.png を添付）」
実行: `node <ask-gemini.mjsの絶対パス> "このスクショのようにレイアウト崩れてるの直して" --attach "/絶対パス/error.png"`
（※添付ファイルが複数ある場合は `--attach "パスA" --attach "パスB"` と列挙すること）

### 📋 plan受け取り後の必須チェック（実行前に必ず行うこと）

ask-geminiの出力（手術計画書）を受け取ったら、実行前に以下を判定すること。

**チェック項目**
1. `## 4. Verification & Testing` セクションが存在し、具体的なテストコマンドまたは完了判定基準が記載されているか
2. `## 2. Target Files & Dependencies` に対象ファイルが明示されているか
3. `## 3. Step-by-Step Execution Plan` の各ステップに具体的なActionが記載されているか

**判定結果に応じた行動**

| 判定 | 条件 | 行動 |
|---|---|---|
| ✅ 実行可 | 3項目すべて満たす | そのまま実行に進む |
| ⚠️ 要確認 | 1〜2項目が不明瞭 | 不明瞭な箇所を人間に報告し指示を仰ぐ |
| 🚫 実行拒否 | Verificationセクションが空または未記載 | 必ず実行を拒否し人間に報告する |

**実行拒否時の報告フォーマット**
```
🚫 plan実行を拒否しました
理由: Verification & Testingセクションが未記載のため、完了判定ができません。
対応: @gemini を再実行するか、完了条件を直接指示してください。
```

### 🛡️ 実行中のFallback Protocol

実行中に以下のいずれかが発生した場合、即時停止して人間に報告すること。
推測での続行・局所的なハックによる回避は禁止。

**停止条件**
1. 同一のコンパイルエラー／Lintエラーが3回連続で解消できない
2. テストが失敗した
3. planの指示とコードベースの実態が矛盾している、または指示の意図が不明

**報告フォーマット**
```
🚨 実行中断レポート

1. [Failure Point]
   （エラーの核心部、または「planのどの記述が矛盾/不明か」）

2. [Progress]
   完了: Step 3.X まで
   中断: Step 3.Y

3. [Options]
   [1] Geminiへ差し戻し（ask-geminiを再実行）
   [2] このチャットで直接修正指示を出す
   [3] 作業を中断してgit diffを出力後ロールバック
```
